# MechBox (MechCalc) 架构演进与下一代改进计划 (纯本地化与重火力演进版)

> **文档定位**：本文档基于 MechBox 当前的实际代码实现（Vue 3 + Electron + SQLite + Solver 引擎），从资深工业软件架构师的视角，总结已落地的架构重构成果，并针对“100% 纯本地化、超大容量知识库、极致丰富功能”的重度辅助设计软件目标，提出下一阶段（v2.0+）的深度缺陷剖析与演进方案。

---

## 1. 核心计算引擎：从“单点求解”向“本地系统级耦合”演进

### 1.1 已落地成果回顾
当前系统已成功在 `src/renderer/engine/solver.ts` 中引入了 **约束求解层 (Solver Layer)**。包含牛顿-拉夫逊法（Newton-Raphson）、二分法及基于惩罚函数的离散优化器。这彻底改变了早期的纯函数“单向映射”，实现了带多约束条件的反向寻优（如轴承选型、螺栓选型等逆向求解）。

### 1.2 当前架构缺陷：缺乏系统级耦合与高级概率分析
**痛点**：
*   **孤岛计算**：目前的求解器仍局限于“单零件”。在实际工业设计中，轴的受力弯曲挠度会直接影响齿轮的载荷分布系数，同时改变轴承的受力分布。这些物理量是强耦合的。
*   **单维度的公差分析**：现有的公差分析通常基于极值法（Worst-case），缺乏基于统计学的批量生产概率预估。

### 1.3 下一代演进方案（v2.0）
1.  **构建本地装配体系统级求解器**：建立跨模块的系统级上下文（System Context）。支持“轴-齿轮-轴承”的传动系统级建模，引入传递矩阵法或轻量级 1D-FEM 在本地求解轴的变形挠度，并自动将结果反哺给轴承和齿轮模块。
2.  **本地超高频 Monte Carlo (蒙特卡洛) 模拟**：在纯本地环境中，利用 Web Worker 或底层 Rust 核心进行数百万次的概率统计模型抽样算法，准确预测批量生产的装配良品率，从而放宽非关键公差，降低制造成本。这完全不依赖云计算，极致榨取本地 CPU 性能。

## 2. 数据中心：打造“超大规模本地工程知识库” (100% 离线)

### 2.1 已落地成果回顾
已成功接入 `better-sqlite3`（见 `src/main/db/database.ts`）。实现了本地标准库的按需 SQL 查询，摒弃了低效的 JSON 全量常驻内存模式，为容纳海量数据打下了地基。

### 2.2 当前架构缺陷：数据量较少与深度不足
**痛点**：
*   **资料库单薄**：目前内置的标准件（如轴承型号、螺栓等级）规格覆盖面不足，缺乏美标（ASME）、德标（DIN）、日标（JIS）的全面收录。
*   **高级材料与选型数据匮乏**：缺乏高级疲劳寿命分析所需的 S-N 疲劳曲线数据点、应力-应变非线性材料属性等深度工程资料。

### 2.3 下一代演进方案（v2.0）
**坚守“纯本地构建”原则，拒绝依赖外部 API，将 MechBox 打造为断网环境下的百科全书。**
1.  **内嵌全量全球工业标准库**：大幅扩充本地 SQLite 数据库，收录涵盖 GB/ISO/DIN/JIS/ASME 的全量标准件尺寸、公差、物理参数。即使在毫无网络的深山车间或保密军工企业，依然能查阅所有必需的工业资料。
2.  **集成“重火力”工程材料数据库**：在本地数据库中构建多维度的材料物性库。不仅包含基础的屈服/抗拉强度，还应涵盖不同温度下的弹性模量衰减曲线、热膨胀系数非线性变化表、常见材料的 S-N 疲劳曲线及化学兼容性矩阵，供求解器随时调用。
3.  **本地知识库扩展与自定义 (Local Extension)**：提供图形化界面，允许工程师将自己公司内部的“非标件手册”、“企业特有材料”一键导入到 SQLite 的 `user_standards` 表中，形成每个工程师、每个保密单位独一无二的本地军火库。

## 3. 跨软件协同：纯本地 CAD 原生插件与双向联动

### 3.1 已落地成果回顾
目前能够将计算结果一键导出为 SolidWorks VBA 宏、FreeCAD Python 脚本或 STEP 组装建议说明，初步打通了从计算数据到 3D 建模的第一道壁垒。

### 3.2 当前架构缺陷：单向生成的体验局限
**痛点**：宏代码生成本质上是“一次性单向抛出”。如果工程师随后在 CAD 中微调了尺寸，MechBox 的参数不会自动更新，容易导致“两层皮”。

### 3.3 下一代演进方案（v2.0）
1.  **研发原生 CAD 本地插件 (Add-in)**：为 SolidWorks / Inventor / SolidEdge 等主流软件开发轻量级的原生插件面板，直接安装在用户的电脑上。
2.  **Local API 与 WebSocket 本机通信**：MechBox 桌面端作为**本地服务端**开启 WebSocket 服务（仅监听 `localhost`）。CAD 本地插件与其建立双向通信通道：工程师在 CAD 中点选特征，MechBox 实时捕获参数并执行合规性校核；在 MechBox 中点击“应用优化”，CAD 中的 3D 模型特征立即通过本地指令重构更新（Live Rebuild），全程无需一丝互联网连接。

## 4. UI/UX 交互：本地 3D 渲染与参数化可视

### 4.1 已落地成果回顾
基于 Vue 3 + Vite + Pinia 构建了具有现代工程软件质感的界面，成功分离了渲染进程与计算引擎。

### 4.2 当前架构缺陷：缺乏直观的空间几何反馈
**痛点**：参数输入过于抽象。组合参数是否会导致结构干涉（如沟槽过深切穿轴壁），往往只能等导入 CAD 后才能发现。

### 4.3 下一代演进方案（v2.0）
1.  **内嵌 WebGL (Three.js / Vue TresJS)**：在本地离线包中集成 3D 渲染引擎。当用户调整螺栓规格或沟槽尺寸时，右侧实时通过本地参数化算法生成并渲染 3D Mesh 模型（数字孪生），实现“所见即所得”。
2.  **本地实时干涉检查**：利用 WebGL 和本地空间计算算法进行包围盒或网格碰撞检测。在软件界面的 3D 场景中，将 O 型圈过度挤压或齿轮根切的受损区域用红色网格高亮警示。

## 5. 极致丰富功能扩展：机械设计的“本地瑞士军刀”

**基于丰富的本地资料，衍生出更多高级本地功能模块：**
1.  **本地失效分析与诊断系统**：内置基于专家规则树（Expert System）的失效诊断工具。用户输入失效现象（如：齿面点蚀、轴承发热），系统在本地匹配出可能的原因及改进建议。
2.  **工况模拟与图表生成**：基于本地计算引擎，允许用户输入一个区间的变量（例如：转速从 1000 变到 5000 RPM），自动在本地计算并在本地图表库绘制出寿命/载荷/温度的 2D/3D 变化云图，帮助工程师寻找最佳工作点。
3.  **综合单位与工程物理计算器**：不仅限于普通的“毫米转英寸”，内置复杂的运动学、动力学、热力学、流体力学常数和公式推导计算器。

## 6. 跨平台与底层：Electron 架构下的 Rust 性能榨取

### 6.1 当前状态与局限
目前采用 `electron-vite` 搭配纯 TypeScript 构建，开发效率极高，但在处理海量本地数据检索和复杂的本地系统级耦合运算时，Node.js (V8) 的单线程与计算性能瓶颈会显现。

### 6.2 架构演进（v3.0 预研）：Rust 核心 + Electron/Vue UI
放弃推翻现有生态去完全重写 Tauri，而是在**保持当前 Electron + Vue 3 极佳工程生态的前提下**，将繁重的计算任务平滑下沉到 Rust。
1.  **Rust 核心下沉 (通过 napi-rs 或 WebAssembly)**：
    *   **高密计算重构**：未来将矩阵运算、系统级优化求解、蒙特卡洛模拟以及海量的本地 SQLite 模糊匹配检索统统用 Rust 重写。
    *   **无缝集成**：利用 `napi-rs` 将 Rust 编译为 Node.js 原生扩展（`.node` 文件），在 Electron 主进程中直接高速调用；或编译为 WebAssembly (WASM)，在渲染进程的 Web Worker 中全速运行。这种混合架构能在不改变 UI 框架的前提下，让本地离线计算达到原生级别的极致性能。
2.  **本地内存极限优化**：
    *   通过 Rust 直接管理大型标准数据的内存映射（Memory Mapping），或者操作 SQLite 底层，绕过 Node.js 的 V8 垃圾回收（GC）与序列化开销。让 MechBox 成为真正意义上“资料全包、运算极速、功能强悍”的纯本地重型核武。

## 7. 市场竞品对标与突围差异化策略

结合目前市面主流工业辅助软件的现状，MechBox 必须打出“降维打击”与“差异化体验”：

### 7.1 对标传统表格插件（如 MITCalc）
*   **市场现状**：MITCalc 功能极其全面，但重度依赖 Excel，UI 停留在上世纪 90 年代，交互极其繁琐，且极易出现单元格公式被误删的工程事故。
*   **MechBox 突围**：提供完全独立的现代客户端体验。计算底层黑盒化保护（Rust核心），但计算过程**白盒化展示**（界面上通过 LaTeX 优雅渲染公式推导过程，一键导出用于 ISO 9001 设计评审的标准 PDF 计算书）。

### 7.2 对标云端 SaaS 工具（如 MechaniCalc）
*   **市场现状**：界面现代，但高度依赖互联网，在军工、保密企业车间或无网现场完全瘫痪；采用高昂的按月订阅制，数据有泄露风险。
*   **MechBox 突围**：主打 **100% 本地离线安全**与**极速瞬时响应**。绝不让用户的一张图纸、一个参数流向外网，直击保密研发单位的核心痛点。

### 7.3 对标顶级巨头软件（如 KISSsoft、Romax）
*   **市场现状**：绝对权威，极其专业。但价格极为昂贵（动辄几十万授权费），学习曲线极其陡峭，对于日常小项目估算来说完全是“杀鸡用牛刀”。
*   **MechBox 突围**：主打**“工程师的快反特种兵”**。不盲目追求深度的有限元网格精度，而是提供基于国家/国际标准的**快速定性与边界估算**，以极简交互让工程师在 1 分钟内得出 80% 准确度的可行方案。

### 7.4 对标大厂自带工具（如 SKF、Parker 官网计算器）
*   **市场现状**：数据准，但存在严重的“厂商锁定”（Vendor Lock-in）。SKF 的工具绝对查不到 NSK 的数据，无法给采购提供多源参考。
*   **MechBox 突围**：做**“中立的工程聚合平台”**。在同一个选型界面下，横向拉齐 SKF、NSK、FAG、MISUMI 的同规格型号与参数进行平行对比，甚至允许导入“企业内定物料代码”，彻底打破大厂的信息壁垒。

---

## 结语：工业软件的终极壁垒
MechBox 的核心护城河不在于“连接云端”，而在于**“断网状态下的绝对可靠、资料的浩瀚与计算的全能”**。未来的 MechBox 将通过“多物理场/系统级耦合求解”、“超大规模本地工程百科知识库”、“与 CAD 的本机双向热链接”以及“参数化 3D 孪生反馈”，成为机械工程师在任何严苛或保密环境下最值得信赖的离线桌面指挥中心。
